# Project Merger

Графическое приложение на Python (PySide6) для объединения файлов проекта в единый Markdown-документ с сохранением структуры и поддержкой `.gitignore`.

## Возможности

### Новое в v3
- **Очистить секреты** — галочка рядом с кнопками сборки. Маскирует в итоговом
  документе токены (AWS, GitHub, Slack, OpenAI, Google, Stripe, Telegram, JWT),
  пароли в присваиваниях, логины/пароли в URL, Bearer-токены, приватные ключи
  и e-mail адреса (`***REDACTED***` / `***EMAIL***`). После сборки выводится
  отчёт: сколько секретов и в каких файлах замаскировано («Show Details»).
  Внимание: автоматическая очистка не даёт 100% гарантии — просматривайте
  документ перед публикацией.
- **Язык интерфейса RU/EN** — кнопка «⚙ Настройки», переключение мгновенное,
  без перезапуска.
- **Лимит размера файла** (в настройках) — файлы больше N КБ не встраиваются
  в документ, вместо содержимого вставляется пометка.
- **Недавние проекты** — список последних папок в верхней панели.
- **Копировать в буфер** — кнопка в диалоге после сборки: весь документ сразу
  в буфер обмена (удобно вставлять в чат с LLM).
- **Оценка токенов** — после сборки показывается примерный объём документа
  в LLM-токенах.


- Выбор корневой папки проекта через GUI (drag & drop или диалог)
- Автоматическое обнаружение и парсинг `.gitignore` (библиотека `pathspec`)
- Создание `.gitignore` с типовыми шаблонами при его отсутствии
- Древовидный просмотр файлов с чекбоксами для ручного выбора
- Фильтрация двоичных файлов (с запасным определением кодировки через `chardet`)
- Генерация Markdown-файла с подсветкой синтаксиса по расширению
- **Сканирование и сборка в фоновых потоках (QThread)** — интерфейс не зависает, отмена работает в любой момент
- **Исключённые папки (`node_modules/`, `venv/` и т.д.) отсекаются на этапе обхода** — сканер в них не заходит
- Стандартные правила исключений обновляются вместе с приложением; пользовательские добавления и удаления хранятся отдельно
- Сохранение последних директорий и настроек в `~/.project_merger_config.json`
- Логирование в консоль и файл `project_merger.log` (подробный режим: `--debug`)

## Структура проекта

```
project-merger/
├── i18n.py                  # Слой переводов RU/EN (без зависимостей от Qt)
├── main.py                  # Точка входа: настройка логирования и запуск приложения
├── ProjectMerger.pyw        # Запуск на Windows без окна терминала (pythonw)
├── config.py                # Настройки (JSON): директории, правила исключений + миграция
├── gitignore_handler.py     # Поиск, создание и парсинг .gitignore через pathspec
├── sanitizer.py             # Очистка секретов: правила-детекторы + маскирование
├── scanner.py               # Быстрый обход ФС (os.scandir) с отсечением исключённых папок
├── merger.py                # Сборка Markdown: дерево, определение бинарных файлов, кодировки
├── requirements.txt         # Зависимости: PySide6, pathspec, chardet
├── install.bat              # Установщик для Windows
├── install.sh               # Установщик для Linux
├── ui/                      # Пакет графического интерфейса
│   ├── __init__.py
│   ├── main_window.py       # Главное окно: дерево файлов, чекбоксы, кнопки сборки
│   ├── rules_dialog.py
│   ├── settings_dialog.py   # Настройки: язык интерфейса, лимит размера файла      # Редактор правил исключений (со сбросом к стандартным)
│   └── workers.py           # Фоновые потоки: ScanWorker и MergerWorker (с отменой)
└── README.md                # Этот файл
```

## Установка

### Windows

Запустите `install.bat` двойным щелчком или в терминале:

```cmd
install.bat
```

### Linux

```bash
chmod +x install.sh
./install.sh
```

### Ручная установка

```bash
pip install -r requirements.txt
```

## Запуск

### Windows — без окна терминала (рекомендуется)

Двойной щелчок по **`ProjectMerger.pyw`** — запустится только GUI, без консоли.
Можно создать ярлык на этот файл на рабочем столе. Все сообщения по-прежнему
пишутся в `project_merger.log` рядом со скриптом.

### Из терминала

```bash
python main.py            # Windows
python3 main.py           # Linux
python main.py --debug    # подробное логирование
```

## Зависимости

- **Python ≥ 3.10** (рекомендуется 3.13)
- **PySide6 ≥ 6.5** — графический интерфейс
- **pathspec ≥ 0.11** — парсинг `.gitignore`
- **chardet ≥ 5.0** — определение кодировки (опционально)
